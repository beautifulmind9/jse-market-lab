# Sprint 9 — Definition of Done

Sprint 9 is complete when:

## Functionality
- user can select a ticker
- system generates summary, stats, and behavior insights
- outputs follow defined structure

---

## Clarity
- summary is understandable in one read
- behavior insights are simple and useful
- no explanation requires decoding

---

## Accuracy
- holding-window comparison reflects real behavior
- consistency reflects median vs average relationship
- reliability reflects win-rate behavior

---

## UX
- no scrolling needed to understand output
- no duplication of information
- layout is clean and readable

---

## Language
- plain, natural English
- no system jargon
- no advice language
- consistent with Sprint 8 tone

---

## Testing
- tests pass
- edge cases handled (low data, missing fields)

---

## Review
- manual app review completed
- outputs feel trustworthy
