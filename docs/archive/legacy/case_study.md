# 📈 JSE Decision Support System — Case Study

---

## 🧠 Problem

Retail investors lack structured tools to:

- evaluate trade quality  
- account for transaction costs  
- manage risk  
- allocate capital effectively  

Most decisions are:
- reactive  
- inconsistent  
- not grounded in data  

---

## 💡 Approach

The system was built iteratively across multiple sprints:

### Phase 1 — Signal Detection
- Identify opportunities using median crossover logic  

### Phase 2 — Quality Scoring
- Rank trades using structured criteria  
- Introduce Tier A / B / C classification  

### Phase 3 — Risk Realism
- Integrate:
  - liquidity filters  
  - transaction costs  
  - volatility considerations  

### Phase 4 — Decision Layer
- Add confidence labels  
- Introduce earnings phase awareness  

### Phase 5 — Allocation Engine
- Translate signals into portfolio decisions  
- Enforce capital constraints  
- Introduce deterministic allocation logic  

---

## ⚙️ Key Innovation

A **deterministic portfolio allocation engine** that:

- converts signals into capital decisions  
- enforces real-world constraints  
- produces consistent, explainable outputs  

---

## 📊 System Capabilities

The system now:

- identifies signals  
- evaluates trade quality  
- assesses risk  
- assigns confidence  
- allocates capital  
- outputs a full portfolio plan  

---

## 🚀 Impact

Users move from:

❌ guessing and reacting  
➡️  
✅ structured, rule-based decision-making  

---

## 🧠 Key Insight

> The hardest problem is not identifying trades —  
> it is deciding how to allocate capital under constraints.
