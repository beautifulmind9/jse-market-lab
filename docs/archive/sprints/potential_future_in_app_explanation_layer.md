# Potential Future In-App Explanation Layer

## Purpose
Explore a future product layer that translates dashboard analytics into simple, investor-friendly explanations directly inside the app.

## Why this may matter later
The dashboard already provides:
- analytical outputs
- rankings
- portfolio planning
- analyst views

A future explanation layer could help everyday Jamaican investors understand what the outputs mean without needing to interpret every table manually.

## Potential in-app outputs
- 3–5 key insights from the latest run
- anything surprising or unusual
- common mistakes the current setup or data suggests
- short explanations of selected features and why they exist

## Potential user value
- improves accessibility for beginners
- supports public/demo use
- helps users understand portfolio discipline
- makes the dashboard more educational
- strengthens the product as a decision-support tool

## Not yet committed scope
This is a roadmap possibility, not automatically the next sprint.

It should only move into implementation when the product team explicitly decides:
- the feature belongs inside the app
- the data source for explanation is stable
- acceptance criteria are defined
- explanation outputs can remain faithful to real analytics

## Scope questions to answer before implementation
1. Which data source should drive the explanation layer?
2. Should explanations be generated from latest run outputs, ranked tables, analyst summaries, or portfolio-plan results?
3. What should be shown by default versus on demand?
4. How much interpretation is acceptable before the tool risks overstating conclusions?
5. How should wording differ between Beginner mode and Analyst mode?

## Implementation guardrails
- no invented insights
- no guaranteed-return framing
- no promotional overreach
- simple language without losing accuracy
- must remain aligned with dashboard logic and real outputs

## Decision status
Deferred until explicitly prioritized in backlog and sprint planning.
