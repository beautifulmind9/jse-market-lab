# Sprint 9 — UX Notes

## Objective
Make ticker analysis feel simple, fast, and useful.

---

## Key Principles

### 1. Summary first
User should understand the stock from ONE sentence.

---

### 2. Minimal stats
Only show what matters:
- win rate
- median return
- average return
- best window
- signal count

---

### 3. Behavior over data
Focus on:
- what it means
- not just what it is

---

### 4. No overload
Avoid:
- long paragraphs
- too many metrics
- repeated information

---

### 5. Consistent voice
Match Sprint 8 tone:
- simple
- neutral
- non-technical

---

## Layout

Ticker Analysis tab:

1. Summary (top)
2. Stats (small table)
3. Behavior insights (short bullet list)

---

## Anti-Patterns

- repeating stats in sentences
- explaining internal system logic
- using financial jargon
- long explanations
