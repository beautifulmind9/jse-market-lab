# Post-Sprint 7 Scope Clarification

## Context
After Sprint 7, a public-facing prompt was defined to support communication about the dashboard using outputs such as:
- 3–5 key insights
- anything surprising or unexpected
- common mistakes the data reveals
- a short explanation of one feature

This prompt is useful and aligned with the broader vision of making the dashboard easier to explain publicly.

## Clarification
The prompt was created primarily to support another workflow focused on marketing/public-sharing content.

Because of that, it should not automatically be treated as the next in-app sprint without an explicit scope decision.

## Why this matters
Product work, documentation work, and marketing work often overlap, but they are not the same thing.

Treating them as identical too early can cause:
- backlog confusion
- unclear sprint goals
- mixed acceptance criteria
- implementation work being defined by communication needs rather than product priorities

## Current interpretation
The prompt is currently best understood as a:
- communication aid
- public sharing aid
- marketing support asset

It may later inform an in-app explanation layer, but that should be a deliberate roadmap decision.

## Next step
Decide whether the next priority is:
1. in-app explanation functionality
2. external insight-content generation
3. a phased approach separating the two
