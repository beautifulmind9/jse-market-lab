# Documentation Archive Guide

This archive keeps sprint-era and operational docs accessible without crowding the public product surface.

## Audit and categorization summary

### Keep as public/core docs
- `docs/product_brief.md`
- `docs/feature_breakdown.md`
- `docs/product_decisions.md`
- `docs/iteration_log.md`
- `docs/portfolio_case_study.md`
- `docs/user_flow.md`
- `docs/uat/` (selected examples)

### Keep as supporting/internal docs
- `docs/archive/legacy/overview.md`
- `docs/archive/legacy/case_study.md`
- sprint-level planning and closeout artifacts retained under `docs/archive/sprints/` for traceability

### Archived
- sprint plans, requirements, scope notes, handoffs, UX notes, backlogs, and redundant closeouts moved to `docs/archive/sprints/`
- non-core UAT files moved to `docs/archive/sprints/`

### Merge/replace
- `overview.md` + `case_study.md` -> `docs/portfolio_case_study.md`
- top-level README rewritten to align with current beta product positioning
