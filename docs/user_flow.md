# 🔄 User Flow

---

## 🎯 Purpose

This document outlines how users interact with the system from signal discovery to final decision-making.

---

## 👤 Beginner Flow

A simplified path for users who want guidance:

1. View trade signals  
2. Read warnings (earnings, volatility, liquidity)  
3. Understand confidence level  
4. Review suggested allocation  
5. Decide whether to execute  

---

## 🧠 Analyst Flow

A deeper workflow for advanced users:

1. Explore signal list  
2. Evaluate quality (Tier A / B / C)  
3. Review structure (trend, spread, volume)  
4. Assess risk (earnings, volatility, liquidity)  
5. Analyze allocation logic  
6. Construct portfolio  

---

## ⚙️ Core System Flow

Signal → Quality → Risk → Confidence → Allocation → Decision

---

## 🔁 Decision Layer

The system transitions users from:

❌ “This looks like a good trade”  
➡️  
✅ “This is how much capital I should allocate given constraints”

---

## 🧠 Design Principle

The system is designed to:

- reduce emotional decision-making  
- enforce structured thinking  
- guide users toward disciplined execution
