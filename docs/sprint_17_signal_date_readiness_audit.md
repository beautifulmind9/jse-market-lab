# Sprint 17 — Signal-Date Readiness Audit

## Scope

This audit covers currently available signal-date related fields used by Portfolio and Ticker Analysis surfaces.

## Available signal-date fields

### 1) `date` (canonical market rows)
- **Where seen:** canonical dataset loaded by ingestion (`ingest_dataset`) and used for market-data recency in the app shell.
- **Reliability:** **High for market row dating**, but this is a market observation date, not an explicit trade signal timestamp.
- **Current usage:** dashboard recency labels and optional signal history date rendering.

### 2) `entry_date` (cost-engine trade rows)
- **Where seen:** analyst dataset generated by `build_analyst_dataset` via cost engine.
- **Reliability:** **Moderate** as a proxy for signal timing because it reflects simulated entry construction, not a dedicated stored signal timestamp from the ranking stage.
- **Current usage:** available in analyst rows and can support placeholder timing status.

### 3) `exit_date` (cost-engine trade rows)
- **Where seen:** analyst dataset generated by `build_analyst_dataset`.
- **Reliability:** **Not suitable** for freshness classification; this is a completion timestamp, not a signal-start date.
- **Current usage:** outcome/backtest context only.

## Missing for future Fresh / Active / Late / Stale logic

To support robust timing labels, the pipeline still needs:

1. **Dedicated signal timestamp field** (e.g., `signal_date`) generated at signal/ranking time.
2. **As-of/reference date alignment** for classification (date freshness is measured against).
3. **Trading-calendar aware age calculation** (business/trading days, not raw calendar days).
4. **Clear lineage contract** so Portfolio and Ticker Analysis use the same signal-date source.
5. **Null/partial-data handling rules** (how labels degrade when date data is missing or malformed).

## Sprint 17 decision

Signal timing is shown as a readiness placeholder (`Not yet assessed` / `Signal date unavailable`) rather than implementing Fresh / Active / Late / Stale labels in this sprint.
