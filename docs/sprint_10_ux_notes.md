# Sprint 10 — UX Notes

## Objective
Turn detailed data into something users can actually understand quickly.

---

## Key Principles

### 1. Drilldown ≠ overwhelm
More detail, but still:
- clean
- structured
- readable

---

### 2. Show → then explain

Order:
1. data
2. pattern summary

---

### 3. Group everything

Avoid raw lists.

Always group:
- by window
- by tier
- by volatility

---

### 4. Keep hierarchy

1. Summary insight (top)
2. Key comparisons
3. Detailed tables (optional focus)

---

### 5. No duplication

Sprint 9 = overview  
Sprint 10 = deep dive  

Do not repeat the same sentences.

---

## Layout

Ticker Drilldown:

1. Pattern Summary (top)
2. Holding Window Comparison
3. Tier Performance
4. Volatility Performance
5. Return Distribution
6. Signal Table (last)

---

## Anti-Patterns

- long paragraphs
- repeating same insight in different sections
- too many numbers without explanation
- mixing explanation with raw data
